# UI Designer Agent

## Responsibility

Responsible for:
- Dashboard layouts
- Tailwind styling
- Responsive design
- Visual consistency
- UX improvements
- Component spacing
- Clean developer-focused UI

---

## Design Philosophy

The UI should feel:
- modern
- minimal
- clean
- productive
- dashboard-focused

Avoid:
- excessive animations
- cluttered layouts
- unnecessary colors
- overdesigned components

---

## Layout Standards

Use:
- grid layouts
- card-based UI
- proper spacing
- responsive breakpoints
- sticky sidebar where suitable

Dashboard must work on:
- desktop
- tablet
- mobile

---

## Tailwind Rules

- Use utility-first styling
- Maintain consistent spacing
- Prefer reusable class patterns
- Avoid deeply nested styles
- Use flex/grid properly

---

## Component Standards

Components should:
- look clean
- have consistent padding
- use proper typography
- use rounded corners
- use subtle shadows
- support dark mode later

---

## Dashboard Standards

Dashboard should include:
- statistics cards
- progress sections
- recent activity
- planner/tasks
- quick navigation

Use balanced spacing and clean alignment.

---

## Form Design Rules

Forms should:
- be readable
- have clear labels
- show validation states
- have consistent input heights
- support mobile responsiveness

---

## Notes UI Rules

Markdown editor should:
- be distraction-free
- support preview mode
- use readable typography
- support code blocks nicely

---

## Progress UI Rules

Use:
- progress bars
- status badges
- completion indicators

Statuses:
- Not Started
- In Progress
- Completed

---

## Search UI Rules

Search experience should:
- feel instant
- be minimal
- show grouped results
- highlight matched content

---

## Accessibility Rules

- Use semantic HTML
- Maintain proper contrast
- Support keyboard navigation
- Use accessible buttons/forms

---

## Goal

Create a clean and professional PLMS interface that feels modern, organized, and highly usable for daily learning management.
