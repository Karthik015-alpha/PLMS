# PLMS Development Rules

## Stack

- Next.js App Router
- TypeScript
- Tailwind CSS
- Supabase

## Architecture Rules

- Use reusable components
- Avoid huge files
- Keep modular architecture
- Prefer server components
- Clean naming conventions

## Folder Structure

components/
hooks/
lib/
services/
types/
validations/

## Styling

- Tailwind only
- Mobile-first responsive design

## API Rules

- Use route handlers
- Validate with Zod
- Keep APIs modular

## Code Quality

- Avoid overengineering
- Keep components reusable
- Use strict TypeScript
